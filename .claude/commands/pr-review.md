# PR Review

Review the current BrewDeck changes as a senior backend engineer.

Check:

- Does the change match existing architecture?
- Are controllers thin?
- Are services focused?
- Are repositories clean?
- Are DTOs used correctly?
- Are paginated endpoints returning PageResponse?
- Are validation messages consistent?
- Are tests updated?
- Are integration tests stable?
- Could this break Sonar, JaCoCo, Spotless, or Dependency Check?

Return:

1. Summary
2. Risks
3. Required fixes
4. Optional improvements
5. Suggested Conventional Commit or PR title
