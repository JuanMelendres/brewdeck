# Documentation Review Checklist

## Accuracy

- [ ] Current implementation is described accurately
- [ ] Future behavior is labeled proposed
- [ ] Names, paths, endpoints, and keys are verified
- [ ] Commands are verified or marked unexecuted
- [ ] No fabricated results or approval status

## Completeness

- [ ] Goals and non-goals
- [ ] Main flows
- [ ] Errors and edge cases
- [ ] Security and permissions
- [ ] Testing
- [ ] Deployment and recovery
- [ ] Risks and open questions

## Maintainability

- [ ] Canonical source identified
- [ ] No unnecessary duplication
- [ ] Relative links are valid
- [ ] Related documents are linked
- [ ] Status and ownership are clear when known

## Safety

- [ ] No secrets or sensitive URLs
- [ ] No unsafe production commands
- [ ] Environment assumptions are explicit

## Diff review

- [ ] Only intended documentation changed
- [ ] No source, migration, build, or infrastructure files changed
- [ ] Pre-existing user changes were preserved
