# Go/No-Go Checklist

## Product

- [ ] Scope approved
- [ ] Acceptance criteria satisfied
- [ ] User-visible behavior documented
- [ ] Feature flags defined

## Architecture and compatibility

- [ ] Architecture approved
- [ ] API compatibility understood
- [ ] Database compatibility understood
- [ ] Multi-service order understood

## Implementation and quality

- [ ] Backend complete
- [ ] Frontend complete
- [ ] Integrations complete
- [ ] Tests pass
- [ ] Accessibility reviewed
- [ ] Final PR review acceptable

## Security and maintenance

- [ ] Security review complete
- [ ] Critical findings resolved
- [ ] High findings resolved or accepted
- [ ] Dependency scans reviewed
- [ ] Container scans reviewed
- [ ] Secrets safe

## Performance and reliability

- [ ] Critical journeys reviewed
- [ ] Timeouts and retries safe
- [ ] Capacity risks understood
- [ ] Health and readiness available
- [ ] Incident handoff ready

## Platform and operations

- [ ] Artifacts immutable and identified
- [ ] Environment configuration ready
- [ ] Migration owner identified
- [ ] Deployment sequence documented
- [ ] Smoke tests documented
- [ ] Monitoring ready
- [ ] Rollback or roll-forward viable

## Documentation and communication

- [ ] Changelog
- [ ] Release notes
- [ ] Migration notes
- [ ] Runbooks
- [ ] Known limitations
- [ ] Internal communication
- [ ] Support communication

## Decision

`GO | GO WITH CONDITIONS | NO-GO | BLOCKED`
