## Recently Worked On

- Slice C.4 refresh tokens (full-stack, branch feature/auth-refresh-tokens) — completes Phase 6. Backend: Flyway V11 `refresh_tokens` (user_id FK, `token_hash` VARCHAR(64) UNIQUE hex SHA-256, `expires_at`, `used_at`, `revoked_at`, `created_at`); opaque 256-bit refresh tokens, only the hash stored (reuses `SecureTokens`). `RefreshTokenService` issues/rotates/revokes: rotation is single-use — presenting an already-used or revoked token is treated as reuse and revokes ALL active tokens for that user; `rotate` is the outermost transaction boundary (`@Transactional(noRollbackFor = InvalidRefreshTokenException.class)`) so the reuse-path revocation commits even though the call then throws a 401, while `AuthService.refresh()` stays non-transactional. Public `POST /api/auth/refresh` (200 → `AuthResponse` with a rotated `token` + `refreshToken`; 401 single generic "Refresh token is invalid or has expired" for invalid/used/expired; 400 blank) and authenticated `POST /api/auth/logout` (204; revokes only the presented token, scoped to the principal). `register`/`login` responses gain a trailing `refreshToken` field (non-breaking). Config: `brewdeck.auth.token-ttl` default shortened `PT24H` → `PT15M`; new `brewdeck.auth.refresh-ttl` default `P7D` (`AUTH_TOKEN_TTL`/`AUTH_REFRESH_TTL` env overrides). Frontend: refresh token persisted in localStorage; silent single-flight refresh on 401 (gated on non-`/refresh` path + a stored refresh token, so no infinite loop); logout now revokes server-side (sign out locally first, then best-effort server revoke). Docs/openapi/Postman/roadmap synced. Two real bugs the review loop caught+fixed: a nested-`@Transactional` rollback that was silently reverting the reuse revocation (fixed by making `rotate()` the tx boundary), and a logout that cleared the access token before the authenticated revoke call so `/logout` went out unauthenticated and never revoked (fixed by clearing storage in a `finally`, with a real-`fetch` regression test asserting the bearer header). Merged to develop (PR #76, all CI green), then released develop→master (PR #77, merge commit; develop kept default).
